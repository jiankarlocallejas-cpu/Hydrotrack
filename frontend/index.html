1. System Overview

HydroTrack is a predictive software tool designed to estimate household water consumption using Multiple Linear Regression (MLR). It processes household and consumption data to generate forecasts, aiming to help families and policymakers improve water usage awareness and sustainability.

2. Core Features

2.1 User Data Input Module

  • Household size
  • Number/type of water-using appliances
  • Frequency of outdoor water use (gardening, car washing, etc.)
  • Daily/Monthly water meter readings
  • Optional: income level, seasonal/weather conditions

2.2 Data Management

  • Store household data and water consumption logs
  • Clean and validate inputs (check for missing/invalid values)
  • Support import/export (CSV, Excel)

2.3 Statistical Analysis

  • Run chosen Multiple Linear Regression model on input variables
  • Identify significant predictors of water consumption
  • Display model performance metrics (RÂ², MAE, RMSE)

2.4 Prediction Engine (optional)

  • Forecast daily/monthly water consumption
  • Show predicted water bill (based on PPCWD rates) (optional)
  • Allow comparison between actual vs. predicted usage

2.5 Visualization & Reporting

  • Graphs for usage trends (daily, monthly)
  • Variable importance chart (which factors affect water use most)
  • Printable/exportable reports for households and barangay councils

2.6 User Interface

  • Web-based preferred (mobile-friendly)
  • Simple dashboards for households
  • Admin/analyst view for researchers (PPCWD or barangay councils in future)

2.7 Security & Ethics

  • Data anonymization for privacy
  • Secure login system
  • Compliance with ethical standards (informed consent, confidentiality)

3. Technical Requirements

  • Frontend: HTML/CSS/JavaScript (React or Vue optional)
  • Backend/API: Python (Flask or Django)
  • Database: SQLite/MySQL/PostgreSQL for structured data storage
  • Data Analysis: pandas, scikit-learn, statsmodels (model must be modifiable / replaceable)
  • Visualization: Chart.js / Recharts / Matplotlib
  • Deployment: Local server or cloud (AWS/GCP/school server)

4. Workflow

  1. Data Collection: Household survey + water meter logs
  2. Data Entry: User inputs or uploads dataset
  3. Processing: Clean and structure the data
  4. Model Training: Run MLR, determine coefficients & significant variables
  5. Prediction: Generate household water consumption forecast
  6. Visualization: Display graphs and reports
  7. Feedback: Provide recommendations for water conservation
